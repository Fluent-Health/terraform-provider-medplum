# medplum_fhir_data_migration (Resource)

Runs an idempotent, self-limiting bulk code-remap over live FHIR resources via Medplum batch/transaction bundles. A task resource: it records that migration `<name>` ran at a given transform hash; it does not track drift of the migrated data.

This is a *task* resource: it records that a named migration ran at a given transform hash, rather than tracking drift of the migrated data. `Create`/`Update` run a converging scan → remap → tag → write loop; `Read` is inert; and `Delete` removes the record from state without reverting data or removing the marker tags. It is idempotent — every scanned resource is stamped with a `meta.tag` marker (`<marker_system>/<name>` = the spec hash) and the scan excludes already-tagged resources (`_tag:not`), so re-running converges and an unchanged configuration plans as a no-op. Changing the transform (a new hash) re-scans everything tagged at the old hash.

Because FHIR has no search parameter for a coded answer value, the scan narrows by the caller-supplied `search` and matches codings in memory. Large migrations are chunked into one bundle per `page_size`. With the default `bundle_type = "batch"`, a page in which any entry fails to write halts the apply (the failed resources are not tagged and would otherwise be re-scanned indefinitely); fix the cause and re-run `terraform apply` to resume — already-migrated resources are skipped.

`import` is not supported: there is no external key to import (the resource is a synthetic record of a migration run).

## Example Usage

```terraform
# Rewrite stored QuestionnaireResponse answers from old numeric diet codes to
# new semantic codes. Run this after the ValueSet/CodeSystem that defines the
# codes has been updated, so the stored data stays consistent with it.
#
# The migration is idempotent: every scanned resource is marked with a meta.tag,
# the scan skips already-marked resources, and re-running converges. An
# unchanged configuration plans as a no-op.
resource "medplum_fhir_data_migration" "diet_codes" {
  name                 = "diet-numeric-to-semantic"
  target_resource_type = "QuestionnaireResponse"

  # Narrow the scan. FHIR has no search parameter for a coded answer value, so
  # the provider searches by an indexable field and matches codings in memory.
  search = "questionnaire=DietIntake"

  code_remap {
    from = { system = "http://example.com/diet", code = "1001" }
    to   = { system = "http://example.com/diet", code = "breakfast" }
  }

  code_remap {
    from = { system = "http://example.com/diet", code = "1002" }
    to   = { system = "http://example.com/diet", code = "lunch" }
  }

  # One bundle per page; default is 50.
  page_size = 100

  # Tip: use `depends_on` to order the migration after the terminology resource
  # that introduced the new codes, e.g.
  #   depends_on = [medplum_fhir_resource.valueset_diet]
}
```

## Schema

### Required

- `name` (String) Stable identity of this migration; also the marker-tag path segment. Changing it forces replacement.
- `search` (String) Raw FHIR search query narrowing the scan, e.g. `questionnaire=A,B`. The provider appends a `_tag:not` self-limiting filter and `_count`.
- `target_resource_type` (String) FHIR resource type to scan and rewrite, e.g. QuestionnaireResponse.

### Optional

- `bundle_type` (String) `batch` (default; per-entry, no rollback) or `transaction` (per-page atomic; requires the project's `transaction-bundles` feature).
- `code_remap` (Block List) Code remaps applied to every Coding in each scanned resource (recursively). At least one is required. (see [below for nested schema](#nestedblock--code_remap))
- `marker_system` (String) Base URI for the `meta.tag` marker system. The tag written is `<marker_system>/<name>` = `<spec_hash>`.
- `page_size` (Number) Resources scanned per page; each page becomes one bundle.

### Read-Only

- `changed_count` (Number) Resources whose codes were rewritten in the last run.
- `failed_count` (Number) Bundle entries that failed in the last run.
- `id` (String) The ID of this resource.
- `scanned_count` (Number) Resources scanned in the last run.
- `spec_hash` (String) Hash of the transform spec; the marker-tag code and the value that drives re-runs.
