# medplum_project_secret (Resource)

A single named entry in the session project's `Project.secret[]` settings. Bots receive all project secrets at execution time as `event.secrets`. Every entry is an independent Terraform resource, so unmanaged sibling entries are preserved; writes to the shared Project resource are guarded by optimistic concurrency (`If-Match` on the project version) with bounded retries, making parallel applies of many secrets safe.

Project secrets are the Medplum way to hand configuration and credentials to bots: every entry in `Project.secret[]` is available to bot code as `event.secrets["<name>"].valueString`. This resource manages one named entry, so a module can own exactly the secrets it introduces while other entries — hand-managed or owned by other configurations — are preserved.

Because all secrets live on the one Project resource, concurrent applies contend on it. Every write here is a read-modify-write guarded by `If-Match` on the project's version, retried with jitter on conflict, so applying many `medplum_project_secret` resources in parallel is safe.

Creating a secret whose name already exists in the project fails rather than silently adopting the existing entry — import it instead.

## Example Usage

```terraform
# A named secret in the session project's Project.secret[] settings. Bots
# receive every project secret at execution time as event.secrets.
resource "medplum_project_secret" "webhook_token" {
  name         = "WEBHOOK_TOKEN"
  value_string = var.webhook_token
}

# Each entry is its own resource: many secrets can be applied in the same run
# (writes to the shared Project resource are serialized with optimistic
# concurrency), and entries not managed by Terraform are left untouched.
resource "medplum_project_secret" "smtp_password" {
  name         = "SMTP_PASSWORD"
  value_string = var.smtp_password
}

# A bot reading the secrets:
#   exports.handler = async (medplum, event) => {
#     const token = event.secrets["WEBHOOK_TOKEN"].valueString;
#     ...
#   };
```

## Schema

### Required

- `name` (String) Secret name, unique within the project (the key bots see in `event.secrets`). Changing it replaces the resource.
- `value_string` (String, Sensitive) The secret's string value (`ProjectSetting.valueString`).

### Read-Only

- `id` (String) The ID of this resource.
- `project_id` (String) Project the secret lives in. Always the provider session's project.

## Import

Import is supported using the following syntax:

```shell
# medplum_project_secret is imported by the secret's name (the entry key in
# the session project's Project.secret[] array).
terraform import medplum_project_secret.webhook_token WEBHOOK_TOKEN
```
