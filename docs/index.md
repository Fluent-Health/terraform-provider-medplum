# Medplum Provider

A Terraform provider for [Medplum](https://www.medplum.com), an open-source FHIR R4 healthcare platform. Use it to declaratively manage FHIR resources, project structure, access policies, client applications, and user memberships — the same objects you would otherwise manage through the Medplum app or API.

## Resources

### FHIR data

- [`medplum_fhir_resource`](./resources/fhir_resource.md) — any FHIR R4 resource (ValueSet, Questionnaire, etc.)
- [`medplum_fhir_profile`](./resources/fhir_profile.md) — FHIR R4 StructureDefinition (custom profile), with Medplum enforcement analysis at plan time

### Project structure

- [`medplum_project`](./resources/project.md) — Medplum project (requires super-admin credentials)
- [`medplum_project_membership`](./resources/project_membership.md) — bind a user or client application to a project with an access policy

### Access control

- [`medplum_access_policy`](./resources/access_policy.md) — fine-grained resource-level access policy
- [`medplum_client_application`](./resources/client_application.md) — OAuth client application with optional OIDC identity provider

### Users

- [`medplum_user`](./resources/user.md) — Medplum user account

## Authentication

The provider supports three authentication modes; pick one:

| Mode | Arguments | Environment variables |
|---|---|---|
| Client credentials (recommended for automation) | `client_id`, `client_secret` | `MEDPLUM_CLIENT_ID`, `MEDPLUM_CLIENT_SECRET` |
| Bearer token (short-lived CI) | `access_token` | `MEDPLUM_ACCESS_TOKEN` |
| Super-admin email/password | `email`, `password` | `MEDPLUM_EMAIL`, `MEDPLUM_PASSWORD` |

Set `base_url` (or `MEDPLUM_BASE_URL`) to the address of your Medplum server.

## Quick start

```terraform
terraform {
  required_providers {
    medplum = {
      source  = "Fluent-Health/medplum"
      version = "~> 1.0"
    }
  }
}

provider "medplum" {
  base_url      = "https://medplum.example.com"
  client_id     = var.medplum_client_id
  client_secret = var.medplum_client_secret
}

resource "medplum_fhir_resource" "example_value_set" {
  resource_type = "ValueSet"

  body = jsonencode({
    resourceType = "ValueSet"
    url          = "https://example.com/fhir/ValueSet/blood-types"
    name         = "BloodTypes"
    status       = "active"
    compose = {
      include = [{
        system = "http://snomed.info/sct"
        concept = [
          { code = "112144000", display = "Blood group A" },
          { code = "165743006", display = "Blood group B" },
          { code = "46251000",  display = "Blood group AB" },
          { code = "58460004",  display = "Blood group O" },
        ]
      }]
    }
  })
}
```

## Example Usage

```terraform
terraform {
  required_providers {
    medplum = {
      source  = "Fluent-Health/medplum"
      version = "~> 1.0"
    }
  }
}

# Option 1 (recommended for automation): OAuth client credentials.
# Set MEDPLUM_CLIENT_ID and MEDPLUM_CLIENT_SECRET in the environment, or inline:
provider "medplum" {
  base_url = "https://medplum.example.com"

  client_id     = var.medplum_client_id
  client_secret = var.medplum_client_secret

  # Pin the Medplum server version so the profile-enforcement analysis
  # uses the correct support matrix (default: 5.0.10).
  medplum_version = "5.0.10"
}

# Option 2: pre-obtained bearer token (e.g. from a CI secrets store).
# provider "medplum" {
#   base_url     = "https://medplum.example.com"
#   access_token = var.medplum_access_token
# }

# Option 3: super-admin email + password (required for medplum_project).
# provider "medplum" {
#   base_url = "https://medplum.example.com"
#   email    = var.medplum_email
#   password = var.medplum_password
# }
```

## Schema

### Optional

- `access_token` (String, Sensitive) Pre-obtained bearer token. Env: MEDPLUM_ACCESS_TOKEN.
- `base_url` (String) Medplum (or gateway) base URL. Env: MEDPLUM_BASE_URL.
- `client_id` (String) OAuth client id. Env: MEDPLUM_CLIENT_ID.
- `client_secret` (String, Sensitive) OAuth client secret. Env: MEDPLUM_CLIENT_SECRET.
- `email` (String) Super-admin email. Env: MEDPLUM_EMAIL.
- `fhir_path` (String) FHIR base path. Default /fhir/R4. Env: MEDPLUM_FHIR_PATH.
- `medplum_version` (String) Medplum server version used to select the profile support matrix. Default 5.0.10. Env: MEDPLUM_VERSION.
- `password` (String, Sensitive) Super-admin password. Env: MEDPLUM_PASSWORD.
- `token_url` (String) OAuth token endpoint. Default base_url + /oauth2/token. Env: MEDPLUM_TOKEN_URL.
